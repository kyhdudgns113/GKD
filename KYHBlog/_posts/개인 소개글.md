### **파일, 폴더 구조**

<div class="block_red_transparent">

#### **index.ts 적극 활용**

- 한 폴더에서 export 할 것들은 전부 index.ts 에서 export 함

- import 할 때는 index.ts 에서 export 한것들만 import 하도록 규칙을 만듦

</div>

<div class="block_yellow_transparent">

#### **Decorator 활용**

- import 하는 파일의 위치에 따라서 경로가 제각각이 될 수 있음

    + 이는 코드 가독성을 저해함

- 공통적으로 쓰이는 경로들은 TypeScript 설정에서 Path 로 지정해두어 Decorator 처럼 쓸 수 있게함

</div>

<div class="block_green_transparent">

#### **FSD 구조 활용 및 용용**

- 알파벳 뒷 순서의 폴더는 자기보다 앞 순서인 폴더를 import 할 수 있음

- 알파벳 앞 순서의 폴더는 자기보다 뒷 순서의 폴더를 import 하지 않도록 함

- FSD 구조와 다른점, 응용한점

    + FSD 구조는 하나의 컴포넌트에 대하여 3개 정도의 깊이까지 내려가며, 옆으로 늘어나는걸 최대한 막는다.

    + 응용버전은 하나의 큰 컴포넌트에 대하여 정말 가급적 깊이를 높이지 않았다.

    + 알파벳마다 폴더 이름을 정해두고 사용했다

        * addon, block(or button), component, element, feature, group

        * module, objects, parts(or page), row, subPage, template, unit...

        * 적절한 이름을 찾지못해 생략된 알파벳들이 있다.

</div>
